Run the API.py code. Take a screenshot of the terminal output. What command did you use to compile and run the code?

I am using a Python 3 build on a MacOS system. To run this file on the terminal, I had to cd my way to the current directory, and then name my file. I named it api.py. This is the command I used to run the Flask server on the terminal:

When running this command, the user Ann is retrieved and her data returned by the server.

{
	"name": "Ann",

	"occupation": "Doctor"

Since Adam is not hard-coded on the users list, the user could not find his data, so the get function returned a 404 server code and “user not found” message.

What capability is achieved by the flask library?

The flask library allows the client to perform the CRUD operations on the users dataset, and receive responses based on the type of request that was sent.

Architecture Evolution Activity. Based on your reading this week, could you write a section that might be appended to this paper, Salah et al, 2016, which would present the next phase of evolution history, from microservices to the technologies which are commonly in use today?

Following on the latest trends, where monolithic and centralized services are losing its capability to cope with the swarming volumes of data interchanged by an ever-increasing number of powerful computing devices. According to Winkfield, Hu and Hoppa (2018) microservice oriented architectures improve the efficiency of the Software Development Lifecycle (SDLC) and waterfall models have been replaced by modern agile DevOps. This is because nowadays elastic scaling is key and handling many software releases annually (usually more than 100) has made this possible. Alongside this, the developer has adopted a more integral role, with security integrated in the SDLC.

To respond to the mentioned challenges that have arisen with the microservices architectures like a very high upfront cost to deploy the required hardware as well as maintainability and right amount of resources to meet the application’s demands. Cloud providers usually have security measures implemented on the architectural model, easing off on the load of the developer and are more scalable and maintainable. These services charge for the amount of hardware used, maximizing the efficiency.
