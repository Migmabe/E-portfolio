Unit activities

These are the main differences between the system that the team designed and the actual system that I developed

The system that I ended up presenting was pretty similar to what the team defined in the System Design document. The CRUD operations were allowed and the system had capability to check for oxygen levels, and cabin temperature and pressure. The user also had the chance to modify their own data, which fulfilled the GDPR requirement. Passwords were stored in hashed format using hashlib with a small twist, salting. Ultimately, the security ON and OFF setting was kept as a major requirement. I modified a few things adding a GUI login window, which actively checked password input and the capability to register new users. In terms of users using the system, I created two types of user credentials with different restrictions when interacting with the system, these were “user” with the least privileges and “admin”. As for data storage, aside from using lists, tuples and dictionaries, I wanted to implement a relational database for complex quering with the benfit of it being a lot more scalable.

Since there was no open discussion in the forum for this topic, I will present the discussion myself and come to a conclusion. As already mentioned on the previous' unit activity, there are major advantages to using microkernels, and industry trends align to this.

Biggs, Lee and Heiser (2018), conclude that microkernel-based systems reduce dramatically the severity of the attacks proving that software structure plays a major role in system security. The percentage of prevented vulnerabilities they found for merely the microkernel architecture is 29%

This study further builds on the fact that microkernels give a much better performance in different fields nowadays, and security being one of them make this architecture even more appealing.
