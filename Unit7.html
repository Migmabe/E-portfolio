Unit activities

Ontology comes across as a rather abstract and complex topic to grasp. In metaphysics, is a branch in philosophy that studies existence and reality (Hoffman and Kumar, 2020). This topic is very big, so ontology “breaks it down” into smaller parts and proposes a set of rules to study reality.

In computer science, ontology translates to the explanation of the context of a domain. This can be a business architecture, web data exchange… but in a machine understandable and human-readable format. There are standards as to how we define, entities, classes, relationships in these domains, and we created some languages to represent ontologies. The different components of the defined area have a human-readable description and an axiom to limit interpretation (Gruber, 1993).

Now, with this introduction, the paper by Arnaut, Oliveira and Lima (2010), collates two of the main ontology languages used in industry into a Service Oriented Architecture (SOA). So far, this technology was mainly used in organising repositories, but in this paper is applied with a twist as they propose a way to track the registering of the service itself in the SOA. Services which are scalable and independent to one another, and composed of artifacts (Requirements, Design, Implementation and Test).

In my ISS system implementation, ontology can be simply put into practice by defining the domain where the classes are used and what interface I will be defining for the different components of my system to seamlessly interact with one another. Every module in the system will be formally described with a statement as to what input it takes, the output and what its functionality is (Paul and Ghosh, 2006). Therefore, ontology can be used to define these exactly, a set of constraints for input-output and the required testing strategy to achieve that.

This is the proposed code that will achieve the ADD, LIST, EXIT and help. The latter is a built-in method from the cmd module:

								
from cmd import Cmd
import os

class Prompt(Cmd):

    intro = "Welcome! Let's do some operations. Enter 'help' to display the available commands"
    prompt = ">>"

    os.system(f"chmod 740 ")

    def do_LISTS(self, arg):
        """Display current directory content."""
        content = os.listdir()
        for n in content:
            #   Protect my files with the chmod command to avoid tampering
            os.system(f"chmod 740 {n}")
            print(n)

    def do_EXIT(self, arg):
        """Exit the application"""
        print("Bye")
        return True

    def do_ADD(self, args):
        """Adds numbers. Please enter them with a space in between"""
        n = 0
        for _ in args.split():
            n += int(_)
        print(n)

Prompt().cmdloop()
print("Shell exited")

Main Security Vulnerabilities with my shell and recommendation to increase its security:

The main one is exposing the directory contents, so the attacker could try access and tamper these if the input is not properly validated. However my code protects the files in the directory by running a “chmod 740” command through every file. This protects the file like this:

  • User: read, write and execute
  • Group: read
  • Others: none

This leads to the other concern which is mainly injection. Every time we accept user input, we must assert its correctness, for instance when calling the ADD method, they could craft input that tries deleting a file in the terminal. However we could clean it by ensuring the right formatting of the input. In this case, I am forcing the input to be converted to integer.
